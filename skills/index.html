스킬이란?

스킬(Skill)은 Claude Code에 등록하는 폴더입니다. 폴더 안에는 필수 파일인 SKILL.md 하나와, 필요에 따라 템플릿, 예시 파일, 스크립트 등 어떤 파일이든 함께 넣을 수 있습니다. Claude는 이 폴더의 내용을 참고해서 작업을 수행합니다.

용어: .md 파일 (마크다운)
스킬 파일의 확장자는 .md입니다. 마크다운(Markdown)은 텍스트에 간단한 서식을 넣을 수 있는 파일 형식으로, 메모장으로도 열어서 편집할 수 있습니다.

스킬의 실행 방식

스킬은 두 가지 방식으로 실행됩니다.

스킬 실행 방식
수동 실행
사용자가 /스킬이름 을 직접 입력해서 실행
자동 실행
Claude가 스킬의 설명(description)을 보고, 현재 작업에 해당 스킬이 필요하다고 판단하면 스스로 로딩

CLAUDE.md와의 차이

CLAUDE.md vs 스킬 — 로딩 방식 비교
CLAUDE.md
대화가 시작될 때 항상 전체 내용이 로딩됨
스킬
평소에는 이름과 설명만 로딩됨. 실행될 때만 본문이 로딩됨

따라서 항상 적용되어야 하는 내용은 CLAUDE.md에, 특정 작업을 할 때만 필요한 내용은 스킬에 넣습니다.

스킬의 3단계 로딩

스킬은 처음부터 전부 읽지 않습니다. Claude가 한 번에 처리할 수 있는 텍스트 양에는 한계가 있으므로, 필요한 만큼만 단계적으로 읽어옵니다.

Progressive Disclosure — 3단계 로딩
1단계
이름 + 설명만 읽기
항상 로딩 · 스킬 1개당 약 100토큰
2단계
SKILL.md 본문 읽기
스킬이 실행될 때만 로딩 · 최대 5,000토큰 권장
3단계
첨부 파일 읽기
필요할 때만 로딩 · 용량 제한 없음

스킬을 100개 등록해도 1단계에서는 이름과 설명만 읽으므로 부담이 적습니다. 실제로 실행되는 스킬만 2단계, 3단계로 넘어갑니다.

1

기존 스킬 찾아보기

스킬을 직접 만들기 전에, 이미 공개된 스킬이 없는지 먼저 찾아보세요. 완벽히 같은 스킬이 아니더라도, 비슷한 스킬을 찾아서 수정하는 것이 더 빠릅니다.

스킬 모음

스킬을 찾을 수 있는 곳
skills.sh — 스킬 디렉토리 사이트. 검색, 인기순 정렬
anthropics/skills — Anthropic 공식. 스킬 표준 스펙 + 레퍼런스 스킬
anthropics/claude-plugins-official — Anthropic 공식 플러그인 저장소
awesome-claude-code — 스킬, 훅, 플러그인 종합 목록
awesome-agent-skills — 1,000+ 스킬. 여러 AI 도구에서 사용 가능
antigravity-awesome-skills — 1,340+ 스킬 컬렉션

Claude에게 조사 시키기

위 사이트를 직접 돌아다니는 대신, Claude에게 조사를 시킬 수도 있습니다.

~한 기능을 하는 스킬이 이미 있는지 skills.sh, GitHub 등에서 조사해줘.

Claude가 여러 사이트를 검색해서 비슷한 스킬을 찾고 비교해줍니다.

2

스킬 파일 구조 이해하기

스킬은 결국 폴더 안에 들어있는 텍스트 파일입니다. 핵심 파일은 SKILL.md 하나이고, 필요하면 보조 파일을 옆에 둡니다.

스킬 디렉토리 구조
.claude/skills/ └── weekly-report/ ← 스킬 이름 = 폴더 이름 ├── SKILL.md ← 필수. 스킬의 본체 ├── template.md ← 선택. 양식, 참고 문서 등 ├── scripts/ │ ├── validate.sh ← 선택. 셸 스크립트 │ └── analyze.py ← 선택. Python 등 어떤 언어든 가능 └── examples/ └── sample.md ← 선택. 예시 파일

scripts/ 안의 파일은 컨텍스트에 로딩되지 않고 실행만 됩니다. 스크립트의 출력 결과만 Claude에게 전달되므로 컨텍스트를 절약할 수 있습니다.

SKILL.md 파일의 구조

SKILL.md는 두 부분으로 나뉩니다: 머리말(frontmatter)과 본문.

용어: 머리말 (frontmatter)
파일 맨 위에 --- 두 줄 사이에 적는 영역입니다. 스킬의 이름, 설명 같은 정보를 여기에 넣습니다. 그 아래가 본문으로, 실제 지침 내용을 자유롭게 작성합니다.

--- (머리말 시작) name: weekly-report description: 주간보고서를 작성합니다. 매주 금요일 보고서 작성 시 사용합니다. --- (머리말 끝) # 주간보고서 작성 스킬 ## 작업 절차 1. 이번 주 완료된 작업 목록을 정리합니다. 2. 다음 주 계획을 작성합니다. 3. template.md 양식에 맞춰 보고서를 생성합니다. ## 주의사항 - 수치 데이터는 반드시 출처를 명시합니다. - 보고서 길이는 A4 1장 이내로 유지합니다.

머리말(frontmatter) 주요 필드

필수 vs 선택 필드
필수
name — 스킬 이름. 영문 소문자, 숫자, 하이픈만 사용
description — 이 스킬이 뭘 하는지 + 언제 쓰는지. Claude가 이걸 보고 자동 실행 여부를 판단
자주 쓰는 선택 필드
allowed-tools — 이 스킬이 사용할 수 있는 도구 목록 (Bash, Read, Write 등)
user-invocable — false로 설정하면 /이름 메뉴에서 숨겨짐. Claude는 여전히 자동으로 로딩 가능
argument-hint — /이름 뒤에 뭘 입력해야 하는지 안내 문구 (예: [issue-number])
context — fork로 설정하면 별도의 독립된 Claude 세션에서 실행 (메인 대화에 영향 없음)
model — 이 스킬 실행 시 사용할 AI 모델 (예: claude-sonnet-4-6)

저장 위치

스킬을 어디에 저장하느냐에 따라 적용 범위가 달라집니다.

스킬 저장 위치와 적용 범위
~/.claude/skills/
→ 내 모든 프로젝트에서 사용
.claude/skills/
→ 이 프로젝트에서만 사용. 팀원과 공유 가능
플러그인 경유
→ 플러그인 설치한 곳에서 사용

용어: ~ (틸드)
경로에서 ~는 홈 폴더를 뜻합니다. Windows에서는 C:\Users\사용자이름 에 해당합니다.

3

플랜 모드로 설계하기

스킬을 만들 때 가장 중요한 원칙:

바로 구현하지 마세요. 계획부터 세우세요.

Claude에게 "~한 스킬 만들어줘"라고 바로 시키면 요구사항이 빠지거나 구조가 비효율적일 수 있습니다. 플랜 모드에서 계획을 먼저 세우고, 검토를 반복한 뒤, 구현을 시작하세요.

워크플로우

플랜 모드 → 구현 워크플로우
Step 1 — 참고 자료 전달 + 플랜 모드 진입
↓
Step 2 — 계획 수정 요청 (반복)
↓
Step 3 — 문제점 검토 요청 (반복)
반복
↓
Step 4 — 구현 시작

단계별 설명

Step 1 — 참고 자료 전달 + 플랜 모드 진입

공식 모범 사례 문서와, 섹션 1에서 찾아둔 비슷한 기존 스킬의 링크를 함께 전달합니다. 그리고 플랜 모드로 계획을 세워달라고 요청합니다.

이 공식 가이드를 참고해서 ~한 기능을 하는 스킬을 만들고 싶어. https://platform.claude.com/docs/ko/agents-and-tools/agent-skills/best-practices 그리고 이 기존 스킬도 참고해: https://github.com/xxx/비슷한-스킬-링크 플랜 모드로 계획부터 세워줘.

플랜 모드 진입 방법
— 위 예시처럼 자연어로 "플랜 모드로 해줘"라고 요청
— /plan 슬래시 커맨드 입력
— Shift + Tab 단축키로 모드 전환

어떤 방식이든 동일하게 플랜 모드로 진입합니다. 이 모드에서는 실제 파일을 생성하거나 수정하지 않고 계획만 세웁니다.

Step 2 — 계획 수정 요청 (반복)

Claude가 작성한 계획서를 읽어봅니다. 계획 아래에 선택지가 나타나는데, 선택지 3번에 수정 사항을 적고 Enter를 누릅니다.

(선택지 3번에 수정 요청 작성) 데이터 출처 부분이 빠져있어. 어디서 데이터를 가져올지도 계획에 포함해줘.

한 번에 완벽한 계획이 나오는 경우는 드뭅니다. 수정을 반복하세요.

Step 3 — 문제점 검토 요청 (반복)

계획이 어느 정도 완성되었다고 느껴져도, 한 번 더 검증합니다.

(선택지 3번에 검토 요청 작성) 이 계획에서 예상되는 문제점을 파악하고, 혹시 추가로 빠진 부분은 없는지 검토해줘.

이 단계에서 예외 상황, 에러 처리, 빠진 케이스 등이 발견됩니다. 괜찮다 싶을 때까지 Step 2, 3을 반복합니다.

Step 4 — 구현 시작

계획에 더 이상 문제가 없으면, 선택지 1번을 선택합니다. 1번은 "Yes"로 시작하는 선택지로, 이걸 누르면 Claude가 계획대로 실제 파일을 생성합니다.

왜 이렇게까지 해야 하나요?

Claude는 구현을 시작하면 빠르게 파일을 생성합니다. 잘못된 구조로 만들어진 뒤에 수정하려면 처음부터 다시 만드는 것과 비슷합니다.

계획 단계에서 충분히 검토하면 구현 후 재작업을 줄일 수 있습니다.

4

공식 모범 사례

Anthropic 공식 문서에서 권장하는 핵심 원칙들을 요약합니다. 전문은 공식 문서에서 확인하세요.

핵심 원칙 5가지

Best Practices Summary
1. 간결하게 쓰세요
일반적인 프로그래밍 지식은 Claude가 이미 알고 있습니다.
Claude가 알 수 없는 것만 적으세요 — 내부 양식, 특정 절차, 도메인 고유 규칙.
2. 자유도를 조절하세요
높은 자유도 — "자연스러운 문체로 작성해" (여러 방법이 다 괜찮을 때)
중간 자유도 — 대략적인 순서와 구조를 제시 (선호하는 패턴이 있을 때)
낮은 자유도 — 정확한 스크립트와 명령어를 지정 (틀리면 안 되는 작업)
3. 점진적 공개(Progressive Disclosure)
SKILL.md에는 개요와 핵심 절차만 쓰세요.
상세한 참고 자료, 템플릿, 예시는 별도 파일로 분리하고 필요할 때 읽게 합니다.
4. 검증 루프를 넣으세요
"작업 후 결과를 확인하고, 문제가 있으면 수정해"라는 단계를 포함하세요.
검증 단계 하나가 출력 품질을 크게 올립니다.
5. 설명(description)을 잘 쓰세요
Claude는 description을 보고 이 스킬을 자동 실행할지 결정합니다.
"무엇을 하는지"와 "언제 사용하는지"를 반드시 함께 적어야 합니다.

피해야 할 패턴

— 본문이 500줄 이상으로 길어지는 것 (별도 파일로 분리)
— 참고 파일이 여러 단계로 중첩되는 것 (SKILL.md → a.md → b.md → c.md)
— 시간에 따라 바뀌는 정보를 고정값으로 직접 적어두는 것
— 설명이 너무 모호해서 Claude가 언제 써야 할지 모르는 것

5

실전: 스킬 만들어보기

지금까지 배운 내용을 합쳐서, 처음부터 끝까지 스킬을 하나 만들어 보겠습니다. 예시로 "회의록 정리 스킬"을 만들어 봅니다.

시나리오

— 회의 후 메모나 녹취록을 Claude에게 주면
— 정해진 양식으로 회의록을 정리하고
— 액션 아이템을 추출해서 담당자별로 분류

Step 1 — 기존 스킬 조사

회의록을 정리해주는 스킬이 이미 있는지 skills.sh, GitHub 등에서 조사해줘.

비슷한 스킬이 있으면 링크를 확보해둡니다.

Step 2 — 참고 자료 전달 + 플랜 모드 진입

이 공식 가이드를 참고해서 회의록 정리 스킬을 만들고 싶어. https://platform.claude.com/docs/ko/agents-and-tools/agent-skills/best-practices 이 기존 스킬도 참고해: https://github.com/xxx/비슷한-스킬-링크 요구사항: - 회의 메모를 받아서 양식에 맞게 정리 - 액션 아이템을 담당자별로 분류 - 양식은 template.md에 저장 플랜 모드로 계획부터 세워줘.

Step 3 — 계획 검토 반복

(수정 요청) 회의 참석자 목록도 자동으로 추출하게 해줘. 날짜와 회의 제목도 메모에서 찾아서 넣어줘. (검토 요청) 이 계획에서 예상되는 문제점을 파악하고, 혹시 빠진 부분은 없는지 검토해줘. (문제없을 때까지 반복)

Step 4 — 구현

계획에 문제가 없으면 선택지 1번(Yes)을 눌러 구현을 시작합니다. Claude가 아래와 같은 파일들을 생성합니다:

생성되는 파일 구조
.claude/skills/ └── meeting-notes/ ├── SKILL.md ← 핵심 지침 └── template.md ← 회의록 양식

스킬을 새로 만들거나 수정한 뒤에는 Claude Code를 재시작해야 합니다.
이미 실행 중인 세션에서는 새 스킬이 인식되지 않습니다.

완성된 스킬 사용법

(Claude Code에서 이렇게 사용) /meeting-notes 오늘 회의 내용: - 김 대리가 API 연동 완료했다고 함 - 박 과장이 다음주까지 디자인 시안 준비하기로 - QA 일정은 4월 셋째주로 확정 - 서버 비용 절감 방안 논의, 이 부분은 추가 조사 필요

이렇게 입력하면 Claude가 스킬의 양식과 규칙에 따라 깔끔한 회의록을 자동으로 생성합니다.

스킬은 수정 가능합니다.

사용 후 결과가 기대와 다르면 SKILL.md나 template.md를 수정하면 됩니다. "이 스킬에서 ~부분을 수정해줘"라고 Claude에게 요청하면 해당 파일을 편집합니다.
