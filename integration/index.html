API란?

API(Application Programming Interface)는 프로그램끼리 데이터를 주고받기 위한 통로입니다.

예를 들어, Jira에서 이슈 목록을 가져오고 싶다면 Jira 화면을 직접 열지 않아도 API를 통해 데이터만 받아올 수 있습니다. Claude Code가 외부 서비스와 소통할 때도 이 API를 사용합니다.

API를 지원하는 서비스라면 Claude Code에서 연결할 수 있습니다.

인증 정보가 필요한 이유

API는 아무나 사용할 수 있는 것이 아닙니다. 외부 서비스에 접근하려면 "이 요청이 허가된 사용자로부터 온 것"임을 증명해야 합니다.

이 증명에 사용되는 것이 인증 정보입니다.

인증 정보의 종류
API 키
서비스가 발급하는 고유 문자열. 비밀번호와 비슷한 역할
토큰
일정 기간 유효한 인증 문자열. 만료되면 갱신 필요
ID + 비밀번호
계정 정보를 직접 사용하는 방식. 보안상 API 키나 토큰이 권장됨

이 인증 정보를 Claude Code에 알려줘야 외부 서비스에 접근할 수 있습니다. 그런데 인증 정보는 민감하기 때문에 코드나 대화에 직접 적으면 안 됩니다.

인증 정보를 안전하게 전달하는 방법이 환경변수입니다.

환경변수에 인증 정보를 저장하면, Claude Code는 실제 값을 모른 채로 변수 이름만 사용해서 API를 호출할 수 있습니다.

2

환경변수로 인증 정보 설정하기

용어: 환경변수 (Environment Variable)
운영체제에 저장되는 이름-값 쌍입니다. 프로그램이 실행될 때 이 값을 읽어서 사용할 수 있습니다. 예: JIRA_API_TOKEN = abc123xyz

설정 파일 위치

Claude Code의 환경변수는 settings.json 파일에서 설정합니다.

설정 파일과 용도
.claude/settings.local.json
내 인증 정보 저장용. Claude Code가 이 파일을 생성하면 자동으로 Git 제외 처리됨
.claude/settings.json
팀 공유 설정용. Git에 포함되므로 인증 정보를 넣으면 안 됨
~/.claude/settings.json
모든 프로젝트에 적용되는 개인 설정

인증 정보는 반드시 .claude/settings.local.json에 넣으세요.
Claude Code가 이 파일을 생성하면 자동으로 글로벌 gitignore에 추가됩니다. .claude/settings.json에 넣으면 Git을 통해 유출될 수 있습니다.

설정 방법

프로젝트 폴더 안의 .claude/settings.local.json 파일에 env 항목을 추가합니다.

// .claude/settings.local.json { "env": { "JIRA_API_TOKEN": "실제_토큰_값", "JIRA_HOST": "https://your-team.atlassian.net", "JIRA_EMAIL": "me@example.com", "SLACK_BOT_TOKEN": "xoxb-실제_토큰_값" } }

이렇게 설정한 뒤 Claude Code를 재시작하면 환경변수가 적용됩니다. 이미 실행 중인 세션에는 반영되지 않으므로, 설정을 변경하면 반드시 재시작하세요.

스킬에서 환경변수 활용

섹션 3에서 만든 스킬과 함께 사용하면, Claude Code가 환경변수의 인증 정보로 외부 서비스에 접근하고 스킬의 절차에 따라 작업을 수행합니다.

스킬 + 환경변수 조합
스킬 (절차)
+
환경변수 (인증)
→
외부 서비스 접근
(예: Jira에서 이번 스프린트 이슈를 가져와 정리하는 스킬) /sprint-report 이번 스프린트 진행 상황 정리해줘.

Claude Code가 환경변수에 저장된 Jira 인증 정보를 사용해서 API로 이슈 목록을 가져오고, 스킬의 양식에 맞춰 보고서를 작성합니다.

3

보안 주의사항

인증 정보는 유출되면 외부인이 해당 서비스에 접근할 수 있으므로 관리에 주의가 필요합니다.

지켜야 할 것
1
인증 정보는 settings.local.json에만 저장
이 파일은 Git에 포함되지 않음
2
대화 중에 인증 정보를 직접 입력하지 않기
환경변수 이름만 알려주면 Claude Code가 알아서 값을 읽음
3
토큰은 만료 기한 확인
토큰 방식은 일정 기간이 지나면 만료됨. 만료되면 새로 발급받아 교체
4
유출 시 즉시 대응
토큰이나 API 키가 유출되면 즉시 폐기하고 새로 발급받아야 함. 비밀번호가 유출되면 즉시 변경
5
설정 변경 후 Claude Code 재시작
환경변수를 추가하거나 수정한 뒤에는 Claude Code를 재시작해야 반영됨
6
.env 파일 접근 차단 설정
settings.json의 permissions에서 민감 파일 읽기를 차단할 수 있음

민감 파일 읽기를 차단하는 설정 예시:

// .claude/settings.json (팀 공유 가능) { "permissions": { "deny": [ "Read(./.env)", "Read(./.env.*)", "Read(./secrets/**)" ] } }
4

MCP — 미리 세팅된 외부 연결

지금까지 설명한 방식은 환경변수에 인증 정보를 넣고 Claude Code가 API를 직접 호출하는 방식입니다.

MCP(Model Context Protocol)는 다른 접근 방식입니다. 외부 서비스와의 연결 로직을 미리 구현해둔 별도의 프로그램(MCP 서버)을 거쳐서 접근합니다.

용어: MCP 서버
"서버"라는 이름이지만, 반드시 원격 컴퓨터에서 실행되는 것은 아닙니다. 내 컴퓨터에서 실행되는 작은 프로그램일 수도 있고, 인터넷의 원격 서비스일 수도 있습니다. 플러그인이나 확장 프로그램에 가까운 개념입니다.

MCP 서버의 두 가지 형태

로컬 vs 원격 MCP 서버
로컬 (stdio 방식)
내 컴퓨터에서 실행되는 프로그램. Claude Code가 시작할 때 함께 실행되고, 종료하면 함께 종료됨
Claude Code
→
로컬 MCP 서버
→
외부 서비스
원격 (HTTP 방식)
인터넷에 있는 서비스. URL로 접속해서 사용. 여러 사람이 동시에 사용 가능
Claude Code
→
원격 MCP 서버
→
외부 서비스

환경변수 방식과의 차이

환경변수 방식 vs MCP 방식
환경변수
Claude Code가 매번 API 호출 코드를 직접 작성해서 실행. 빠르게 시작 가능하지만 결과가 매번 다를 수 있음
MCP
미리 구현된 코드가 실행되므로 항상 같은 방식으로 동작. 일관된 결과를 얻을 수 있지만 MCP 서버 세팅이 필요

MCP 서버 자체는 개발자나 서비스 제공자가 만들어둔 것을 사용합니다. 사용자가 할 일은 그 MCP 서버를 Claude Code에 연결하는 것입니다. 연결 방법은 각 MCP 서버 제공자의 가이드를 따르면 됩니다.

MCP는 표준 프로토콜이므로 연결 절차 자체는 동일하지만, 서버 주소나 인증 방식 같은 설정값은 서버마다 다릅니다. 각 MCP 서버에서 안내하는 대로 연결하는 것이 가장 정확합니다.
